# 開発者がそのまま使える「推奨フォルダ構成」の詳細説明

このセクションでは、Next.js (App Router) を使用したプロジェクトの推奨フォルダ構成について、各ディレクトリ・ファイルの目的、役割、具体的な使用例、およびベストプラクティスを詳細に説明します。これにより、開発者がプロジェクト構造を迅速に理解し、効率的に開発を進められるようになります。

## 主要ディレクトリとファイルの役割

### 1. `public/`

*   **役割**: 静的アセット（画像、フォント、faviconなど）を配置します。Next.jsによって自動的にルートパスとして提供されます。
*   **詳細**: 
    *   直接 `localhost:3000/image.png` のようにアクセス可能です。
    *   画像は `public/images/logo.png` のようにサブディレクトリで整理することを推奨します。
    *   `next/image` コンポーネントを使用する場合も、基本的にはここに画像を配置します。

### 2. `src/`

*   **役割**: アプリケーションの主要なソースコードを格納するルートディレクトリです。Next.jsはデフォルトで `src` ディレクトリをサポートしており、コードの整理に役立ちます。

#### 2.1. `src/app/` (App Router ルートディレクトリ)

*   **役割**: Next.js 13+ のApp RouterにおけるルーティングとUIのコケーション（関連するコードを近くに配置する）の基点となります。
*   **詳細**: 
    *   **ルーティング**: ファイルシステムベースのルーティングを採用しており、`app` ディレクトリ内のサブディレクトリ名がURLパスに対応します。
        *   例: `app/customers/page.tsx` は `/customers` に対応します。
        *   動的ルートは `[slug]` や `[id]` のように角括弧で囲んで表現します。例: `app/customers/[customer_id]/page.tsx`。
    *   **データフェッチング**: デフォルトでサーバーコンポーネントであり、直接データベースやAPIを呼び出すことができます。これにより、クライアントへのバンドルサイズを減らし、パフォーマンスを向上させます。
    *   **レイアウト (`layout.tsx`)**: 複数のルートセグメントで共有されるUIを定義します。例えば、ダッシュボード全体のサイドバーやヘッダーを `app/(dashboard)/layout.tsx` に定義し、そのグループ内の全てのページに適用させることができます。
        *   **ルートグループ (`()`)**: URLパスに影響を与えずに、特定のレイアウトや挙動をまとめるために使用します。例: `(auth)` でログインページ群、`(dashboard)` で認証後のページ群をまとめる。
    *   **ページ (`page.tsx`)**: 特定のルートセグメントのユニークなUIを定義します。各URLパスの最終的なレンダリング結果となります。
    *   **読み込み中UI (`loading.tsx`)**: ルートセグメントのデータフェッチ中など、読み込み中に表示されるUI（例: スケルトンスクリーン、ローディングスピナー）を定義します。
    *   **エラーUI (`error.tsx`)**: ルートセグメントでエラーが発生した際に、フォールバックとして表示されるUIを定義します。ErrorBoundary のような機能を提供します。
    *   **API Routes (`api/`)**: `app/api` ディレクトリ以下に配置されるファイルは、APIエンドポイントとして機能します。Next.jsのサーバーレス関数としてデプロイされます。RESTfulなAPI設計に沿ってCRUD操作などを実装します。

#### 2.2. `src/components/` (UIコンポーネント)

*   **役割**: アプリケーション全体で再利用可能なUIコンポーネントを格納します。関心事を分離し、保守性と開発効率を高めます。
*   **詳細**: 
    *   **`ui/`**: Shadcn/ui、Radix UI、Material UI などのヘッドレスUIライブラリや、それらをラップして独自のテーマやスタイルを適用した汎用的なUIプリミティブ（例: `Button.tsx`, `Input.tsx`, `Modal.tsx`, `Card.tsx` など）を配置します。これらはアプリケーション全体のデザインシステムを構成し、一貫性のあるUIを提供します。
    *   **`common/`**: アプリケーション全体で共通して使用されるレイアウトコンポーネントやナビゲーションコンポーネント（例: `Header.tsx`, `Sidebar.tsx`, `Footer.tsx`, `LoadingSpinner.tsx` など）を配置します。これらは異なるルートグループやページ間で共有されます。
    *   **`[feature_name]/`**: 特定の機能（顧客管理、予約管理、ペットカルテ管理など）に特化したコンポーネントを配置します。これにより、関連するコンポーネントが一箇所にまとまり、見つけやすく、機能ごとの独立性が高まります。例: `customers/CustomerList.tsx`, `customers/CustomerForm.tsx`, `appointments/CalendarView.tsx`, `medical-records/MedicalRecordCard.tsx` など。

#### 2.3. `src/lib/` (ビジネスロジック、ユーティリティ、外部サービス連携)

*   **役割**: アプリケーションのビジネスロジック、データフェッチング、外部サービスSDKの初期化、汎用的なユーティリティ関数などをカプセル化します。再利用可能で、UIやフレームワークから独立したロジックをここに置きます。
*   **詳細**: 
    *   **`supabase/`**: 
        *   `client.ts`: クライアントサイドでのSupabaseクライアント（`createClientComponentClient`）の初期化を行います。主にクライアントコンポーネントからSupabaseにアクセスする際に使用します。
        *   `server.ts`: サーバーサイド（サーバーコンポーネント、API Routes、ミドルウェア）でのSupabaseクライアント（`createServerComponentClient`）の初期化を行います。Next.jsのヘッダーからクッキーを安全に読み込み、認証済みのリクエストをSupabaseに送るために重要です。
        *   `utils.ts`: Supabaseからのデータ変換、共通のクエリビルドロジック、RLSポリシーと連携するためのヘルパー関数など、Supabase関連のユーティリティ関数を配置します。
    *   **`api/`**: Supabaseからのデータフェッチングや書き込みロジックをカプセル化した関数群を配置します。直接API RoutesでSupabaseのテーブル操作を書く代わりに、ここで定義された関数を通じてデータアクセスを行うことで、ロジックの再利用、テストのしやすさ、および将来的なデータソースの変更への対応が容易になります。例: `customers.ts` (`getCustomers`, `createCustomer`), `pets.ts` (`getPetsByCustomerId`, `createPet`)。
    *   **`auth.ts`**: 認証関連のヘルパー関数（現在のセッション情報の取得、ユーザーの役割チェックなど）を定義します。Supabase Authと連携し、認証状態に基づくアクセス制御ロジックを集中管理します。
    *   **`resend.ts`**: Resendクライアントの初期化と、メール送信機能を提供するヘルパー関数を定義します。メールテンプレートのレンダリングロジックなどもここに含めることができます。
    *   **`line.ts`**: LINE Messaging APIクライアントの初期化と、メッセージ送信機能を提供するヘルパー関数を定義します。

#### 2.4. `src/hooks/` (カスタムReactフック)

*   **役割**: Reactのロジックを再利用可能にするためのカスタムフックを配置します。主にクライアントサイドの状態管理、データフェッチング、副作用の処理を行います。
*   **詳細**: 
    *   `useCustomers.ts`: SWRやReact Queryなどのデータフェッチングライブラリを使用して、顧客データをフェッチ、キャッシュ、更新するためのカスタムフック。ローディング状態、エラー状態なども管理します。
    *   `useAuth.ts`: 認証状態（ログイン済みか、ユーザー情報、役割など）をアプリケーション全体で利用するためのカスタムフック。Supabase Authと連携します。
    *   `useForm.ts`: フォームの入力値管理、バリデーション、送信処理などを抽象化する汎用的なフォームフック。

#### 2.5. `src/types/` (TypeScript 型定義)

*   **役割**: アプリケーション全体で使用されるTypeScriptの型定義を集中管理します。これにより、コードの型安全性を確保し、開発効率を向上させます。
*   **詳細**: 
    *   `supabase.ts`: Supabase CLIの `supabase gen types typescript --db-url
