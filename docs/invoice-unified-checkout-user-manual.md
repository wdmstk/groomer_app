# 統合会計 運用マニュアル（店舗向け）

Task ID: `TASK-401`

## 1. マニュアルに必ず記載すること
- 対象業務: 同一顧客のトリミング予約とホテル滞在を1回の会計で確定する。
- 主導線: `/payments` の「統合請求（β）」を優先利用し、予約単位会計は例外時のみ使う。
- 作成導線: `/hotel` の詳細カード「統合会計を作成」から請求（invoice）を起票する。
- 会計確定条件: 会計確定時に予約IDの解決が必要。ホテル滞在の `appointment_id` が未設定だと確定不可。
- 整合条件: `appointment_ids` / `hotel_stay_ids` は同一顧客であること。
- 完了後動作: 会計確定で `payments` が1件生成され、領収書表示・来店履歴連携が可能になる。
- 例外運用: 統合請求に含められない案件のみ「予約単位会計（レガシー）」を使う。

## 2. 具体的な操作方法（現場手順）
1. トリミング予約を作成する
- 画面: `/appointments?tab=list&modal=create`
- 入力: 顧客、ペット、予約メニュー、開始日時、担当スタッフ
- 実行: `登録する`
- 確認: 一覧に予約行が追加される

2. ホテル予約を作成する
- 画面: `/hotel`
- 実行: `新規予約を登録`
- 入力: 顧客、ペット、チェックイン/チェックアウト予定、宿泊明細
- 実行: `作成`
- 確認: 台帳一覧に `stay_code` と料金が表示される

3. カルテを作成する（トリマー作業）
- 画面: `/medical-records?tab=pending&appointment_id=<予約ID>`
- 入力: 記録日、状態メモ、注意事項、必要に応じて写真（施術前/施術後）
- 実行: `登録する`（必要に応じて `status=finalized` で確定）
- 確認: 「下書きを保存しました。」または「カルテを確定しました。」

4. ホテル詳細から統合会計を作成する
- 画面: `/hotel`
- 実行: 対象滞在を選択し `統合会計を作成`
- 確認: 「統合会計を作成しました（請求ID: ...）」

5. 会計管理で統合請求を確定する
- 画面: `/payments?tab=list`
- 実行: `統合請求（β）` で対象請求IDの `会計確定` を開く
- 入力: 支払方法、備考（任意）
- 実行: `確定して領収書へ`
- 確認: 領収書画面（`/receipts/<payment_id>`）へ遷移

## 3. 失敗しやすいポイント
- 「予約IDを解決できません」: ホテル滞在に `appointment_id` が紐付いていない。
- 「INVOICE_SCOPE_MISMATCH」: 予約とホテル滞在の顧客が一致していない。
- 二重計上リスク: 統合対象案件をレガシー会計で先に確定しない。
