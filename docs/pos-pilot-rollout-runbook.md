# POS パイロット展開 Runbook（TASK-POS-006）

## 目的
- POS機能を `pilot -> full` で段階展開し、業務停止リスクを抑えながら本番移行する。

## フェーズ定義
1. Phase 0: ステージング確認
2. Phase 1: パイロット1-2店舗
3. Phase 2: 同業態店舗へ段階拡大
4. Phase 3: 全店舗展開

## Phase 0（ステージング）
1. [POS 移行/初期設定手順](./pos-migration-initial-setup.md) を実施
2. [POS UATチェックリスト](./pos-uat-checklist.md) を全件 `pass`
3. 障害連絡先、切戻し責任者、判断時刻を確定

## Phase 1（パイロット）
1. 対象店舗を1-2店舗に限定
2. 初日と翌日の2営業日を重点監視
3. 監視項目
- `pos_orders` の `confirmed/void` 件数
- `inventory_movements` の POS notes 付き件数
- `cash_drawer_events` と締め差異（`cash_diff`）
4. 判定基準
- 重大障害 0
- 締め差異が運用許容内
- 店舗から継続可の合意

## Phase 2（段階拡大）
1. 業態/規模が近い店舗を3-5店舗ずつ追加
2. 追加前に前フェーズのインシデント是正を完了
3. 週次でKPIレビュー
- POS会計件数
- 取消率
- 在庫差異率

## Phase 3（全店舗）
1. 全店舗へ告知と運用手順配布
2. レガシー導線は1リリース期間併存
3. 併存期間終了後にPOS標準運用へ一本化

## インシデント時対応
1. 症状切り分け（UI / API / DB / RLS）
2. 新規POS会計を停止し、レガシー導線へ切替
3. 既存データは保持し、監査ログを保全
4. 原因と再発防止を記録して再リリース判断

## 完了条件
- パイロット店舗で2営業日連続 `重大障害 0`
- UATと運用手順が最新版へ更新済み
- 運用責任者承認（pilot -> full）が記録済み
