# POS運用マニュアル（店舗スタッフ向け）

Task ID: `TASK-POS-006`

## 1. このマニュアルの目的
- POSの基本操作を、現場スタッフが迷わず実行できるようにする。
- 日々の運用（開局→会計→取消→締め）を統一する。

## 2. 事前準備
1. 画面を開く
- `/payments?tab=list`
- 画面上部の `POS会計（β）` パネルを使う

2. 必要データを確認
- 顧客、予約、商品（在庫商品）が登録済みであること

3. ルール
- 開局前に会計確定しない
- 誤会計の取消は領収書画面から行う
- 営業終了時は必ずレジ締めする

## 3. 基本運用フロー（毎日）
1. 開局する
- `レジセッション` の `開局` ボタンを押す
- 「開局中: 日時」が表示されることを確認

2. 必要なら現金入出金を記録する
- `入出金種別` を選ぶ（入金 / 出金 / 調整）
- `金額` と `理由` を入力して `登録`
- 「現金入出金を登録しました。」の表示を確認

3. POS会計を確定する
- `予約` を選択（必要に応じて顧客が自動反映）
- `顧客` を確認
- `支払方法` を選択
- `商品`、`数量`、`単価(税込)` を入力し `明細追加`
- 必要なら `値引き` を入力
- `会計確定して領収書へ` を押す
- 領収書画面に遷移すれば完了

4. 誤会計を取消する（必要時）
- 領収書画面 `/receipts/<payment_id>` へ移動
- `POS取消` セクションで `取消理由` を入力
- `この会計を取消` を押す
- 取消完了メッセージを確認

5. 締め処理を行う
- `POS会計（β）` パネルへ戻る
- `実残高` と `締めメモ` を入力して `レジ締め`
- サマリ（売上合計 / 現金期待額 / 実残高 / 差異）を確認

## 4. 会計操作の詳細
1. 明細追加
- 同じ商品を複数回追加した場合は別行で積み上がる
- 誤った行は `削除` で削除する

2. 金額表示
- `小計`、`税額`、`値引き`、`合計` が自動計算される

3. 確定できない主な条件
- 開局していない
- 予約未選択
- 顧客未選択
- 明細が0件

## 5. 取消時の注意
1. 取消実行後の動作
- POS伝票は `void` 状態になる
- `payments.status` は `取消` になる
- 物販在庫は自動で戻し処理される

2. 取消理由
- 監査・後追い確認のため、必ず業務上わかる理由を記載する

## 6. 締め処理の見方
1. 売上合計
- セッション内の確定会計の合計

2. 現金期待額
- 現金会計 + 入金 - 出金 + 調整

3. 差異
- `実残高 - 現金期待額`
- 差異がある場合は締めメモに理由を残す

## 7. トラブルシューティング
1. `先にレジを開局してください。`
- 開局が未実施。`開局` ボタンを押して再実行

2. `在庫不足です`
- 商品在庫が不足。数量を減らすか、入庫後に再実行

3. `POS_SESSION_ALREADY_OPEN`
- 既に開局済み。開局操作は不要

4. 取消に失敗する
- 対象会計の状態を確認
- 既に取消済みの場合は重複取消は不要

## 8. 日次チェック（終業前）
- [ ] 未処理の会計がない
- [ ] 必要な取消が完了している
- [ ] レジ締めを実施した
- [ ] 差異がある場合、締めメモに理由を残した
