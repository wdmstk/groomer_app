# POS要件定義（MVP）

Task ID: `TASK-POS-001`  
Parent Task: `TASK-408`

## 1. 目的
- 既存の会計基盤（`payments` / `invoices`）と在庫基盤（`inventory_*`）を活かし、店舗現場が1画面で会計業務を完了できるPOS要件を定義する。
- 先行MVPで「通常会計の安定運用」を実現し、その後フェーズで返品・締め処理を段階導入する。

## 2. 対象業務
- トリミング会計
- ホテル会計
- 店販売上会計
- 上記の混在会計（同一顧客、同一会計内でサービス + 物販）

## 3. 対象ロール
- `owner`: 全操作可（設定、締め、取消、監査閲覧）
- `admin`: 会計登録、返品、日次締めまで可（運用設定は制限）
- `staff`: 会計登録と領収書再発行のみ可（返品/締めは不可）

## 4. MVPスコープ
1. POS会計画面（サービス + 物販の明細作成）
2. 支払方法選択（現金 / カード / QR / 電子マネー / その他）
3. 値引き入力（固定金額）
4. 会計確定時の領収書発行
5. 会計確定時の在庫自動出庫（物販のみ）
6. 会計取消時の在庫戻し（同日かつ未締めセッションのみ）
7. 監査ログ記録（作成 / 確定 / 取消）

## 5. MVP対象外（後続フェーズ）
- 複数支払手段の分割決済
- レジ開閉局と現金過不足精算の本運用
- 高度なポイント/クーポン連携
- 複数税率の自動判定（暫定は明細税率を明示保持）

## 6. 機能要件

### FR-01 伝票作成
- 伝票は顧客単位で作成できる。
- 明細は `service` と `product` を混在可能とする。
- 明細ごとに `quantity` / `unit_price` / `tax_rate` / `tax_included` を保持する。

### FR-02 金額計算
- 伝票小計、税額、合計、値引後合計を画面上で即時計算する。
- 値引は0未満不可、割引後合計は0未満にしない。

### FR-03 会計確定
- 会計確定時に idempotency key を必須とし、重複確定を防ぐ。
- 確定後は領収書画面へ遷移する。

### FR-04 支払方法
- 支払方法は既存会計互換の文言セットを使用する。
- 未選択時は確定不可とする。

### FR-05 在庫連動
- `product` 明細のみ在庫減算対象とする。
- 会計取消時は同量を戻し仕訳する。

### FR-06 取消制御
- 締め済み会計は取消不可（返金フローへ誘導）とする。
- 取消時は理由入力を必須とする。

### FR-07 監査
- 伝票作成、確定、取消の各操作に `actor_user_id`、`store_id`、差分を監査ログ保存する。

### FR-08 マルチストア/RLS
- すべてのPOSデータは `store_id` スコープでアクセス制御する。
- 他店舗データ参照・更新は常に拒否する。

## 7. 非機能要件
- 性能: 会計確定APIはP95で2秒以内（通常負荷）。
- 可用性: 会計確定失敗時に再実行可能（idempotencyで安全担保）。
- 監査性: 金額・在庫に影響する操作は全件追跡可能。
- 可観測性: 失敗率、重複確定回避件数、在庫連動失敗件数をメトリクス化可能。
- セキュリティ: 権限外ロールの取消/締め操作を禁止。

## 8. 既存システム統合方針
- `payments`: 最終的な入金記録は既存テーブルを継続利用する。
- `invoices` / `invoice_lines`: サービス明細との整合維持を優先し、POS明細は拡張 source_type で接続する。
- `inventory_movements`: POS確定で `outbound`、取消で逆仕訳を生成する。
- `receipts`: 既存領収書画面を拡張し、サービス・物販両明細を表示する。

## 9. 受け入れ基準（Given/When/Then）

1. 通常会計（サービスのみ）
- Given 顧客とサービス明細が1件ある
- When 会計確定を実行する
- Then `payments` が1件作成され領収書へ遷移する

2. 通常会計（物販のみ）
- Given 在庫が十分な商品明細が1件ある
- When 会計確定を実行する
- Then `inventory_movements` に出庫が作成される

3. 混在会計
- Given サービス明細と物販明細を同一伝票へ追加した
- When 会計確定を実行する
- Then 合算金額で1件の会計が確定し、物販分のみ在庫が減る

4. 値引き境界
- Given 合計3000円の伝票がある
- When 4000円の値引きを入力する
- Then 割引後合計は0円で確定可能（負値にならない）

5. 支払方法必須
- Given 会計伝票が作成済み
- When 支払方法未選択で確定する
- Then バリデーションエラーで確定されない

6. 二重送信防止
- Given 同一伝票で同一 idempotency key を持つ再送が起きる
- When 2回目の確定処理が到達する
- Then 新規会計は作成されず初回結果が再利用される

7. 他店舗アクセス防止
- Given 別店舗のPOS伝票IDを指定したリクエスト
- When 取得または更新を試行する
- Then 403または404で拒否される

8. 在庫不足時
- Given 在庫数を超える数量の物販明細
- When 会計確定を実行する
- Then 在庫不足エラーで確定されない

9. 会計取消（未締め）
- Given 未締めセッションの会計データ
- When 理由付きで取消を実行する
- Then 会計は取消状態になり在庫戻し仕訳が作成される

10. 会計取消（締め済み）
- Given 締め済みセッションの会計データ
- When 取消を実行する
- Then 取消不可エラーとなり返金導線を案内する

11. 監査ログ
- Given 会計作成・確定・取消を実行した
- When 監査ログを参照する
- Then 操作主体と差分が全件確認できる

12. 領収書再表示
- Given 会計確定済みの伝票
- When 領収書再表示を開く
- Then 初回発行と同一明細・同一金額が表示される
