# 主要画面のUIモック

トリミングサロン向け業務管理システムの主要な画面のUIモックをテキストベースで説明します。具体的なレイアウトや要素、画面遷移のイメージを提供します。

## 1. ログイン画面

*   **目的**: システムへのアクセスを認証する。
*   **レイアウト**: 中央にログインフォーム。
*   **要素**:
    *   サロンのロゴ
    *   「ログイン」タイトル
    *   メールアドレス入力フィールド
    *   パスワード入力フィールド
    *   「ログイン」ボタン
    *   （任意）「パスワードを忘れた場合」リンク
*   **画面遷移**:
    *   ログイン成功: ダッシュボードホーム（顧客一覧または予約カレンダー）へ遷移。
    *   ログイン失敗: エラーメッセージを表示。

```
+-----------------------------------+
|              [ロゴ]               |
|                                   |
|             ログイン              |
|                                   |
|  メールアドレス: [ __________ ]   |
|  パスワード:   [ __________ ]   |
|                                   |
|           [ ログイン ]            |
|                                   |
|       パスワードを忘れた場合      |
+-----------------------------------+
```

## 2. ダッシュボードレイアウト (共通)

ログイン後の主要画面は、共通のレイアウト（サイドバー、ヘッダー）を持ちます。

*   **ヘッダー**: 上部に固定。
    *   サロン名/ロゴ
    *   現在のページタイトル
    *   ログインユーザー名、ログアウトボタン
    *   （任意）通知アイコン、設定アイコン
*   **サイドバー**: 左側に固定。
    *   ダッシュボードホーム
    *   顧客管理 (顧客一覧、新規顧客登録)
    *   予約管理 (予約カレンダー、予約一覧)
    *   ペットカルテ管理 (カルテ一覧)
    *   設定 (スタッフ管理など)

```
+-------------------------------------------------------------+
| [ロゴ] サロン名           [現在のページ]           [ユーザー名] [ログアウト] |
+-------------------------------------------------------------+
|               |                                             |
| サイドバー    |                                             |
| - ダッシュボード|                                             |
| - 顧客管理    |                                             |
| - 予約管理    |             メインコンテンツエリア            |
| - カルテ管理  |                                             |
| - 設定        |                                             |
|               |                                             |
|               |                                             |
+-------------------------------------------------------------+
```

## 3. 顧客一覧画面 (`/dashboard/customers`)

*   **目的**: 登録されている顧客情報を一覧表示・検索・管理する。
*   **レイアウト**: ヘッダー、サイドバー、メインコンテンツエリアに顧客リストと検索・フィルタリングUI。
*   **要素**:
    *   「顧客一覧」タイトル
    *   「新規顧客登録」ボタン
    *   検索バー（氏名、電話番号、メールアドレスなどで検索）
    *   フィルターオプション（LTVランク、タグなどで絞り込み）
    *   顧客情報テーブル/リスト
        *   カラム: 氏名、電話番号、最終来店日、LTVランク、ペット数、アクション (詳細、編集、削除)
    *   ページネーション
*   **画面遷移**:
    *   「新規顧客登録」ボタン: 新規顧客登録フォームへ遷移。
    *   顧客名クリック/「詳細」ボタン: 顧客詳細画面へ遷移。

```
+-------------------------------------------------------------+
| [ロゴ] サロン名           顧客一覧                 [ユーザー名] [ログアウト] |
+-------------------------------------------------------------+
|               |                                             |

| - ダッシュボード|                                             |
| - 顧客管理    |  検索: [ __________ ] [検索]                  |
| - 予約管理    |  フィルタ: [LTVランク▼] [タグ▼]             |

| - 設定        |  +---------------------------------------+  |
|               |  | 氏名  | 電話番号 | 最終来店日 | LTV | ペット数 | アクション |
|               |  |-------|----------|------------|--------|--------|------------|

|               |  | 鈴木花子| 080...   | 2025/12/15 | C | 1匹    | [詳細] [編集] [削除] |
```

## 4. 顧客詳細画面 (`/dashboard/customers/[customer_id]`) 

*   **目的**: 特定の顧客の詳細情報を表示・編集し、関連するペット、予約、来店履歴などを確認する。
*   **レイアウト**: ヘッダー、サイドバー、メインコンテンツエリアに顧客情報とタブ切り替えUI。
*   **要素**:
    *   「顧客詳細」タイトル
    *   「編集」ボタン、削除ボタン
    *   顧客基本情報表示エリア (氏名、住所、電話番号、メール、LINE ID、来店経路、LTVランク、タグ)
    *   タブナビゲーション:
        *   **ペット情報**: 顧客に紐づくペット一覧（名前、犬種、生年月日、性別、アクション: 詳細、編集、削除）
            *   「新規ペット登録」ボタン
        *   **予約履歴**: 過去・未来の予約一覧（日時、メニュー、担当者、ステータス、アクション: 詳細、変更、キャンセル）
        *   **来店履歴**: 過去の来店履歴一覧（来店日、メニュー、担当者、金額）
*   **画面遷移**:
    *   「編集」ボタン: 顧客情報編集フォームへ遷移。
    *   ペット名クリック/「詳細」ボタン: ペット詳細画面へ遷移。
    *   予約クリック/「詳細」ボタン: 予約詳細画面へ遷移。

```
+-------------------------------------------------------------+

| - ダッシュボード|                                             |
| - 顧客管理    |  --- 顧客基本情報 ---                       |
| - 予約管理    |  氏名: 山田太郎                             |
| - カルテ管理  |  住所: 東京都...                            |
| - 設定        |  電話: 090-xxxx-xxxx                        |
|               |  ...                                        |
|               |  --- タブナビゲーション ---                 |
|               |  [ ペット情報 ] [ 予約履歴 ] [ 来店履歴 ]     |
|               |                                             |
|               |  +---------------------------------------+  |
|               |  | ペット名 | 犬種 | 生年月日 | 性別 | アクション |
|               |  |----------|------|----------|------|------------|
|               |  | ポチ     | 柴犬 | 2020/05/01 | オス | [詳細] [編集] [削除] |

```

## 5. 予約カレンダー画面 (`/dashboard/appointments`)

*   **目的**: 日付ごとの予約状況を視覚的に確認・管理する。
*   **レイアウト**: ヘッダー、サイドバー、メインコンテンツエリアにカレンダーと予約詳細表示。
*   **要素**:
    *   「予約カレンダー」タイトル
    *   年月移動ボタン (前月、次月)
    *   表示モード切り替え (日、週、月)
    *   スタッフ選択フィルター
    *   カレンダーグリッド:
        *   日付セルに予約情報を表示（顧客名、ペット名、メニュー、時間、担当者）。
        *   予約をクリックすると詳細情報ポップアップまたは詳細画面へ遷移。
    *   「新規予約登録」ボタン
*   **画面遷移**:
    *   日付セルクリック/「新規予約登録」ボタン: 新規予約登録フォームへ遷移。
    *   予約クリック: 予約詳細画面へ遷移。


## 6. ペットカルテ詳細画面 (`/dashboard/customers/[customer_id]/pets/[pet_id]/medical-records/[record_id]`)

*   **目的**: 特定のペットのカルテ詳細情報を表示・編集し、施術履歴や写真を管理する。
*   **レイアウト**: ヘッダー、サイドバー、メインコンテンツエリアにカルテ情報。
*   **要素**:
    *   「カルテ詳細」タイトル
    *   「編集」ボタン、削除ボタン
    *   ペット基本情報表示エリア (名前、犬種、性別、年齢、体重、ワクチン、持病)
    *   施術履歴セクション:
        *   施術日、担当者、メニュー、所要時間、使用シャンプー、皮膚状態、問題行動、注意事項
        *   （任意）「新規施術履歴追加」ボタン
    *   写真管理セクション:
        *   アップロードされた写真のサムネイル一覧
        *   「写真追加」ボタン (Supabase Storageへのアップロード機能と連携)
        *   写真クリックで拡大表示
*   **画面遷移**:
    *   「編集」ボタン: カルテ情報編集フォームへ遷移。

```
+-------------------------------------------------------------+
| [ロゴ] サロン名           カルテ詳細                 [ユーザー名] [ログアウト] |
+-------------------------------------------------------------+
|               |                                             |
| サイドバー    |  [ 編集 ] [ 削除 ]                             |
| - ダッシュボード|                                             |
| - 顧客管理    |  --- ペット基本情報 ---                     |
| - 予約管理    |  名前: ポチ                                 |
| - カルテ管理  |  犬種: 柴犬                                 |
| - 設定        |  生年月日: 2020/05/01 (4歳)                 |
|               |  体重: 8.5kg                                |
|               |  ...                                        |
|               |                                             |
|               |  --- 施術履歴 ---                           |
|               |  2025/12/25 - 担当:田中 - シャンプーコース - 皮膚状態:良好 ... |
|               |  2025/10/10 - 担当:佐藤 - カットコース - 問題行動:噛みつき傾向 ... |
|               |  [ 新規施術履歴追加 ]                       |
|               |                                             |
|               |  --- 写真管理 ---                           |
|               |  [サムネイル1] [サムネイル2] [サムネイル3]  |
|               |  [ 写真追加 ]                               |

```

これらのUIモックは、主要な画面フローと情報表示のイメージを提供します。実際のデザインは、Next.jsのコンポーネントライブラリ（例: shadcn/ui, Material UI, Ant Designなど）やCSSフレームワーク（例: Tailwind CSS）を使って構築することになります。
